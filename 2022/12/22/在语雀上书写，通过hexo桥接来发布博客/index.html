在语雀上书写，通过hexo桥接来发布博客

1. 前言

终极懒癌，不断在寻找一个比较适合自己的写作解决方案，感觉不解决就无法提笔写作。
市面上相关的一些服务或工具主要分为几个部分：
静态站点生成工具 + mardown 书写工具 + 静态页面托管站的组合。

1.1 常见静态站点管理工具

  1. Jekyll
  2. hexo
  3. hugo

1.2 Markdown 书写工具

  1. Typora
  2. VSCode / Sublime Text / Vim
  3. yuque
  4. notion

1.3 静态页面托管站点

  1. Github pages
  2. Gitee pages
  3. 各种云服务器

不断尝试之后，静态站点管理工具都差不多，编辑器的话 yuque 给我留下了深刻的印象，不仅支持各种插入代码块，资源链接，体验很出色，托管站的话还是熟悉的 Github pages 最香。
接着在发现 yuque-hexo 这个插件之后，就萌生了在 yuque 上编写文章，然后同步文章，交由 hexo （静态站点管理工具）生成静态页面，最后部署到 github pages 的想法。

2. 最终目标

最终写作的流程就是：

  1. 在 yuque 编辑文章并且更新
  2. 通过 yuque 的 webhook 自动调用云服务器的 API，再通过 Github Dispatch 来触发 Github Action
  3. 同步 yuque 的文章，并且生成静态页面，自动部署到 Github Pages


整个时许如上图所示，详细的步骤下面会拆成几个部分来说明：

  1. 创建 Hexo 的应用，存放在 Github Pages 的 Repository
  2. 创建 yuque 的知识库，做相关配置
  3. 配置 yuque-hexo，实现自动同步文章
  4. 创建 Github Action 实现自动同步 yuque 的文章并部署 Github Pages
  5. 创建云服务器来触发第四步所创建的 Github Action，添加 webhook，打通整个流程

3.1 创建 hexo 应用

  1. 安装 hexo 命令行工具

3.2 创建 github repository

  1. 创建 Repository，Repository Name 为 {{ github用户名 }}.github.io

  1. 不需要 README， .gitignore，license，点击 Create repository
  2. 获取该 repo 的 git 地址
  3. 回到本地的 blog 工程，添加远程地址

3.3 配置 yuque 知识库

  1. 打开账户设置

  2. 获取个人路径，账号管理 → 个人路径，保存为 login：{{ path }}

  1. 创建 Token，Token → 新建 → 勾选权限 → 创建，保存为 YUQUE_TOKEN : {{ token }}

  1. 创建新的知识库
  2. 打开知识库的设置

  1. 获取知识库的路径，知识库信息 路径，保存为 repo：{{ path }}

3.4 安装 yuque-hexo 插件并测试

  1. 打开本地的博客的工程，运行命令添加 yuque-hexo 插件
  1. 可以看到 package.json 里依赖项多了对 yuque-hexo
  2. 配置 yuque-hexo，参考 https://github.com/x-cold/yuque-hexo
  3. 编辑 package.json，添加新的 “yuqueConfig”，从上一步 yuque 知识库里获取的 “login” “repo” 配置好
  1. 拷贝上一步获取的 YUQUE_TOKEN，在终端测试一下是否可以同步文章
  1. 如果在终端看到同步的日志，那么就表示同步成功
  1. 测试成功之后，就可以将这些更改同步到 github repo

3.5.1 CI 触发条件

这边表示有两个触发条件：

  1. 当 main 分支发生 push 事件触发的时候
  2. 当 workflow_dispatch 事件触发的时候

至于 workflow_dispatch 事件的话，是为了我们能够通过非 git 事件来触发而添加的。
它支持以下几种方式来触发：

  1. Github API

https://docs.github.com/en/rest/actions/workflows?apiVersion=2022-11-28#create-a-workflow-dispatch-event

  1. Github CLI （gh 命令）

gh workflow run --help

  1. Github browser interface

https://docs.github.com/en/actions/managing-workflow-runs/manually-running-a-workflow

3.5.2 环境变量

可以看到 CI 里面有两个环境变量，secrets.GITHUB_TOKEN 和 secrets.YUQUE_TOKEN。
GITHUB_TOKEN 是官方预留的一个环境变量，凡事采用这个 token 的最后都是由 githhub-actions 的机器人来完成操作的。

YUQUE_TOKEN 则是 yuque-hexo 插件所需要的一个环境变量。
https://github.com/x-cold/yuque-hexo#%E9%85%8D%E7%BD%AE-yuque_token
然后这个环境变量可以按照下面的步骤来添加：

  1. 点击 Repository 的 Settings
  2. 在侧边栏找到 Security → Security → Actions
  3. 点击右边的 New repository secret 按钮
  4. 在弹出的界面里，Name 一栏里面填上 YUQUE_TOKEN，在 Secret 一栏里面填上 3.3 步骤的第三步里拿到的 YUQUE_TOKEN，点击 Add secret 即可。
